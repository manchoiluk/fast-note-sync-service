## v3.1.1
> *2026/05/25*

### 🚀 优化

- **WebGUI**: 优化主题菜单交互。点击主题按钮会弹出一个清爽的菜单，支持「跟随系统 / 自动 / 浅色 / 深色」四种选择，并且按钮图标会根据当前生效的主题自动变换。

## v3.0.6
> *2026/05/23*

### ✨ 新增

- **MCP**: 新增 `file_write` 工具，支持第三方客户端通过 Base64 编码直接写入附件或文件并实时同步至各端（请避免上传超大文件，以防内存不足发生 OOM）。
- **网络**: 新增自定义 `header` 响应头配置，支持在配置文件中为所有 HTTP 服务和 WebSocket 握手阶段自定义响应头（详见 `config.yaml` 中的 `server.custom-response-headers` 选项）。

### 🚀 优化

- **WebGUI**: 优化笔记列表移动交互。支持通过鼠标拖拽移动笔记或附件到根目录或其他子目录。
- **WebGUI**: 优化 Obsidian 属性渲染。笔记编辑页和分享笔记页现在可以正确显示 Obsidian 笔记的属性（Properties）。
- **WebGUI**: 优化分享笔记入口。在笔记编辑页增加一键“分享笔记”的按钮入口。

### 🛠️ 修复

- **自启动**: 修复 macOS 自启动服务因权限不足导致无法正常运行，以及快捷安装脚本执行报错的问题。
- **配置**: 修复配置文件中显式将默认开启的布尔配置（如 `webgui-login-token-bind-ip`, 云存储启用开关及 WebSocket 并行/压缩选项）设为 `false` 后依然被默认值覆盖的问题。
- **WebGUI**: 修复 WebGUI 界面在加载图片或附件资源时，由于浏览器请求未携带特定客户端 Header 导致文件被拦截报错的问题。
- **WebGUI**: 修复 WebGUI 编辑标题显示多余路径的问题。

## v3.0.0
> *2026/05/14*

> [!WARNING]
> **注意事项**：本次更新引入了全新的令牌架构，会导致旧版所有授权失效。升级后您需要对所有客户端设备进行重新授权。

### 🔐 核心更新：全新有状态令牌权限系统

本项目重构了底层的安全架构，从单一无状态令牌升级为 **有状态多维度（协议、客户端、功能）** 校验令牌：
- **多维度校验**：
  - **协议控制**：支持对 `REST API`、`WebSocket`、`MCP` 等不同访问协议及场景进行独立授权与限制。
  - **客户端限制**：支持通过通配符绑定特定设备（如 `obsidian*`），实现“一机一令”。
  - **内容控制**：细化至笔记读写 (`note_r/w`)、附件管理 (`file_r/w`)、系统配置 (`config_r/w`) 等原子级权限。
- **世代校验与令牌轮转 (Rotation)**：
  - 引入 `Nonce` 世代机制。令牌在存在泄露风险时可进行轮转操作，旧世代令牌将立即失效，有效防范令牌被盗用的风险。
- **动态环境绑定**：
  - 支持对令牌进行 **IP 地址** 和 **User-Agent** 的静态或通配符绑定，用于在非安全环境下限制令牌使用行为。
- **令牌访问统计**：
  - 服务端将会记录所有令牌的访问行为，支持实时监控各令牌的调用协议、来源 IP 及客户端版本。

### ✨ 新增

- **备份**: 新增备份任务密码加密设置。全量与增量备份任务新增灵活的备份包加密策略，支持“不设密码、手动固定密码、系统随机密码”三种配置模式，显著提升备份数据的存储安全性。

### 🛠️ 修复

- **云存储**: 升级阿里云 OSS v2 SDK 驱动，修复存储配置连接测试失败及区域识别问题。

### 🚀 优化

- **性能**: 优化文件处理性能。调整哈希计算阈值为 10MB，通过延迟创建物理目录大幅减少文件系统 I/O 开销。
- **协议**: 优化协议标准化。所有 WebSocket 返回消息强制包含 `vault` 字段，确保多库同步状态的一致性。
- **安全**: 安全加固。备份历史记录中的敏感密码字段默认开启掩码处理，支持手动解密查看。
